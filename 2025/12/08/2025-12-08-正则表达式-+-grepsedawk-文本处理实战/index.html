正则表达式 + grepsedawk 文本处理实战

一、正则表达式核心基础

正则表达式（Regular Expression）是用于匹配文本模式的字符串，分为 基础正则（BRE） 和 扩展正则（ERE），常用元字符如下（重点记忆）：

元字符 含义说明 示例
. 匹配任意单个字符（除换行） a.b 匹配 aab、acb、a1b
* 匹配前一个字符 0 次或多次 ab* 匹配 a、ab、abb、abbb
+ 匹配前一个字符 1 次或多次（扩展正则） ab+ 匹配 ab、abb（需 grep -E）
? 匹配前一个字符 0 次或 1 次（扩展正则） ab? 匹配 a、ab
^ 匹配行首 ^hello 匹配以 hello 开头的行
$ 匹配行尾 world$ 匹配以 world 结尾的行
[] 匹配括号内任意一个字符 [a-zA-Z0-9] 匹配字母数字
[^] 匹配括号外任意一个字符（取反） [^0-9] 匹配非数字字符
() 分组（扩展正则），用于提取或替换 (ab)+ 匹配 ab、abab
| 逻辑或（扩展正则） a|b 匹配 a 或 b
{n,m} 匹配前一个字符 n 到 m 次（扩展正则） a{2,3} 匹配 aa、aaa
\d 匹配数字（等价于 [0-9]，部分工具支持） \d{3} 匹配 123、456
\w 匹配字母、数字、下划线（等价于 [a-zA-Z0-9_]） \w+ 匹配用户名、变量名

注意：

grep

默认支持基础正则（BRE），扩展正则需加

-E

参数；

sed

基础正则需转义

()+{}|

，扩展正则加

-E

可免转义；

awk

默认支持扩展正则。

二、三大工具核心用法（结合正则）

1. grep：文本过滤与搜索

grep 核心功能是按正则匹配行，常用参数：

  • -i：忽略大小写

  • -n：显示匹配行号

  • -v：反向匹配（不匹配的行）

  • -E：支持扩展正则

  • -o：只显示匹配的字符串（而非整行）

  • -r：递归搜索目录下所有文件

2. sed：文本替换与编辑

sed 是流编辑器，按正则批量修改文本，常用语法：sed 's/正则/替换内容/选项' 文件名

  • s：替换命令（substitute）

  • 选项：g（全局替换）、i（忽略大小写）、p（显示匹配行）、w（写入文件）

  • 其他常用命令：d（删除行）、a（行后追加）、i（行前插入）

3. awk：文本分析与数据提取

awk 按列处理文本（默认以空格 / 制表符为分隔符），支持正则匹配、条件判断、循环，核心语法：awk '正则 {动作}' 文件名

  • $n：第 n 列（$0 表示整行）

  • FS：字段分隔符（如 FS="," 表示逗号分隔）

  • NR：行号

场景 1：分析 Nginx 访问日志，提取 TOP10 访问 IP

日志格式示例：192.168.1.1 - - [01/Jan/2024:10:00:00 +0800] "GET / HTTP/1.1" 200 1234

四、避坑指南

  1. 正则转义问题：

  • grep 基础正则中，()+{}| 需转义（如 a\+b），加 -E 后无需转义（a+b）

  • sed 基础正则需转义 ()+{}|，扩展正则（-E）免转义

  • Shell 中 $、*、? 是特殊字符，正则中需加反斜杠 \ 转义（如 \$）

  1. 匹配精度问题：

  • 用 \b（单词边界）避免部分匹配（如匹配 80 不匹配 8080）

  • 用 [^"] 匹配非引号字符（如提取 JSON 字段时避免跨字段匹配）

  1. 工具选择原则：

  • 简单搜索 / 过滤 → grep

  • 批量替换 / 删除 → sed

  • 按列提取 / 统计 → awk

  • 复杂场景 → 组合使用（如 grep | sed | awk）
