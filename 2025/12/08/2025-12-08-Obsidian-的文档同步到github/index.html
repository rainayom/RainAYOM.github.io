Obsidian 的文档同步到github

将 Obsidian 文档同步到 GitHub，推荐用Obsidian Git 插件实现自动化同步，比手动输 Git 命令更高效，还支持多设备联动。以下是详细操作步骤，涵盖电脑端核心配置，也包含手机端适配要点：

一、前期准备

  1. 安装必备工具：安装电脑版 Obsidian；安装 GithubDesktopDownload GitHub Desktop | GitHub Desktop；注册并登录 GitHub 账号。
  2. GitHub 创建私有仓库：登录 GitHub 后点击右上角 “+” 选择 “New repository”，填写仓库名（如 “obsidian-notes”），勾选 “Private” 设为私有，可勾选 “Add a README file”，最后点击 “Create repository”。创建后复制仓库的 SSH 或 HTTPS 地址备用。

二、关联本地 Obsidian 与 GitHub 仓库

1.克隆远程仓库到本地：

打开Github Desktop ，左上角file-clone a repository 。选择在GitHub新建的仓库

选择仓库，克隆到本地

2.Obsidian 关联本地仓库：

打开 Obsidian，点击 “打开本地仓库”，选择刚刚克隆的本地仓库文件夹，此时 Obsidian 的笔记就与仓库文件关联了。

.obsidian文件夹就是Obsidian 配置文件

把GitHub\obsidian-notes.obsidian\workspace.json这个文件从git管理的文件中排除出去（即不上传到GitHub中）。

在GitHub\obsidian-notes.obsidian\文件夹的目录新建一个文件名为.gitignore的文件，编辑文件内容

三、测试上传到GitHub中

在GitHubdesktop里选中已经修改的文件，这里会自动勾选。

在summary里填写提交的说明，然后点击commit进行提交

然后点击右上角的Push按钮进行提交

提交成功后，刷新网页的项目会发现提交成功了

四、安装插件实现自动同步

  1. 安装 Obsidian Git 插件：打开 Obsidian，进入设置→社区插件，关闭 “安全模式”，在插件市场搜索 “Git”（作者 Vinzent），点击安装并启用。

进入选项，设置

开启停止编写后1分钟自动提交

启动Obsidian时从GitHub拉取同步：pull on startup

问题1：“Vault is not a Git repo or git binary cannot be found”

说明当前 Obsidian 库不是 Git 仓库，或者系统没找到 Git 程序，按以下步骤逐个解决：

自动提交需要自己的电脑安装git软件,电脑下载安装Git - 下载 - Git 版本控制系统

第一步：确认 Git 是否已安装且能被系统识别

  1. 检查 Git 安装状态：

    • Windows：打开 “命令提示符”，输入git --version，若显示版本号（如git version 2.43.0）则正常；若提示 “不是内部或外部命令”，说明 Git 未安装或未配置环境变量。
    • Mac：打开 “终端”，输入git --version，无输出则需安装 Git（可通过brew install git安装）。

  2. 配置 Git 环境变量（Windows）：

    若 Git 已安装但未识别，找到 Git 安装目录（如

    ），将其添加到系统环境变量的 “Path” 中，重启电脑后重试。

第二步：在插件中指定 Git 程序路径（若仍报错）

若 Git 已安装但插件未识别，手动指定路径：

  1. 打开 Obsidian 设置→第三方插件→GitHub Sync 插件→找到 “Git Path” 选项。
  2. 填写 Git 程序的完整路径：
    • Windows：D:\Program Files\Git\bin\git.exe
    • Mac：/usr/bin/git

完成后重启插件，即可正常自动同步。

assets/${noteFileName}/${generatedAttachmentFileName}
